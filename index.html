PM ARSENAL

Four field tools for the parts of the job the software forgot: seeing risk early, briefing up, balancing load, and holding the room.

DRAFTED BY: HAILEY CUMMINGS
ROLE: SENIOR PROJECT MANAGER
REV: 02

Why this exists. Most PM software treats projects as static checklists — it tracks tasks, not uncertainty. Four gaps show up repeatedly in how modern PM tools fall short: they don't surface risk patterns early, they don't translate project health into language executives actually read, they measure workload by task count instead of real capacity, and they offer no way to practice the hard conversations that used to be learned by shadowing a veteran in the room. This tool addresses each directly, using AI where it adds real judgment (risk analysis, briefing, roleplay) and plain tracking where it doesn't need to be smarter than a spreadsheet (capacity).
